# /write-proposal - 공공 제안서 작성

리더 에이전트를 통해 제안팀 4개 에이전트를 순차 호출하여, RFP 분석부터 기술 제안서, 가격 제안서, 최종 검토까지 공공 제안서 작성 전체 프로세스를 실행합니다.

## 입력

RFP 정보: $ARGUMENTS

## 실행 절차

### Stage 1: RFP 분석 (rfp-analyzer)

RFP 문서에서 평가항목, 배점, 가중치, 필수 규격, 자격요건을 구조화된 형태로 추출합니다.

1. RFP 문서 읽기
2. 평가항목/배점표 추출
3. 필수 규격 및 자격요건 정리
4. 전략 포인트 도출 (고배점 항목 중심)
5. 제안서 목차 구성 (권장)

**산출물**: RFP 분석 결과 리포트
**Gate Review**: 분석 결과 확인 → 사용자 승인

### Stage 2: 기술 제안서 작성 (proposal-writer)

RFP 분석 결과를 기반으로 평가항목별 기술 제안서를 작성합니다.

1. 평가항목 ↔ 챕터 매핑 확정
2. 챕터별 분량 배분 (배점 비례)
3. 챕터별 본문 작성 (도표/다이어그램 포함)
4. 차별화 포인트 강조

**산출물**: `docs/proposals/{프로젝트명}/` 기술 제안서
**Gate Review**: 기술 제안서 초안 확인 → 사용자 승인

### Stage 3: 가격 제안서 작성 (price-proposal)

기술 제안서 내용을 기반으로 가격 제안서와 산출내역서를 작성합니다.

1. 필요 자원 목록 추출 (인력, 장비, SW)
2. 단가 산출 (공표가/시장가 기준)
3. 사업비 산출내역서 작성
4. 가격 경쟁력 분석

**산출물**: 가격 제안서 + 산출내역서
**Gate Review**: 가격 적정성 확인 → 사용자 승인

### Stage 4: 제안서 검토 (proposal-reviewer)

완성된 제안서의 품질을 검토하고 개선점을 제시합니다.

1. 평가항목별 충족도 점검
2. 예상 점수 산출
3. 누락/미흡 항목 식별
4. 형식 준수 여부 확인
5. 개선 권고사항 제시

**산출물**: 검토 리포트 (예상점수, 개선사항)
**Gate Review**: 검토 결과 확인 → 최종 수정 여부 결정

## 사용 예시

```bash
# RFP 파일 경로 지정
/write-proposal docs/rfp/2026-서버구축-RFP.pdf

# RFP 내용 설명
/write-proposal 조달청 서버 인프라 구축 사업, 예산 5억, 기술평가 90점

# 특정 단계부터 시작
/write-proposal --from stage2 docs/proposals/서버구축/
```

## 주의사항

- 각 Stage 완료 후 반드시 사용자 승인을 받습니다
- 제안서는 `docs/proposals/{프로젝트명}/` 디렉토리에 저장됩니다
- 기존 IT 자산/가격 DB 데이터가 있으면 자동으로 활용합니다
- 리더 에이전트는 직접 제안서를 작성하지 않고 서브 에이전트를 조율합니다
